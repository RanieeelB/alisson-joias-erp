# Documentação do Projeto

Este diretório segue uma regra simples de linguagem:

- Informações para IA, agentes e implementação ficam em inglês.
- Informações para o desenvolvedor, apresentador e pessoas que assistirão à apresentação ficam em português.

Os termos técnicos do desafio podem continuar em inglês quando forem nomes do domínio ou da stack, como Invoice, Accounts Receivable, Accounts Payable, Statements, QuickBooks, Supabase e App Router.

## Para Leitura Humana

- `pre-development.md`: visão geral do pré-desenvolvimento.
- `requirements.md`: requisitos do teste organizados por etapa.
- `roadmap.md`: sequência de branches, entregáveis e critérios.
- `presentation-guide.md`: roteiro de apresentação e perguntas prováveis.
- `visual-direction.md`: direção visual do produto.
- `ai-usage-log.md`: registro de uso de IA para demonstrar maturidade no processo.
- `diagrams/README.md`: guia dos diagramas em português.

## Para IA e Implementação

- `../AGENTS.md`: regras principais para agentes.
- `architecture.md`: arquitetura técnica e limites de integração.
- `data-model.md`: modelo de dados e observações de Supabase/Postgres.
- `tasks.md`: backlog operacional para execução por agentes.
- `skills.md`: skills instaladas, pesquisadas e quando acionar cada uma.

## Observação de Confidencialidade

Os PDFs originais do teste não devem ser versionados. Esta documentação resume as decisões e requisitos necessários sem copiar o material confidencial.
